## Overview

The Authentication API allows you to register new users and authenticate existing users. All endpoints return a JWT token that must be included in the `Authorization` header for subsequent API requests.

**Base Path:** `/api/v1/auth`

## Endpoints

### Register User

Create a new user account and receive an authentication token.

<Card title="POST /api/v1/auth/register" icon="user-plus">
  Register a new user with email and password
</Card>

#### Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be a valid email format.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Must be between 6 and 72 characters.
</ParamField>

#### Response

<ResponseField name="token" type="string">
  JWT authentication token to use for subsequent API requests
</ResponseField>

#### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST https://api.example.com/api/v1/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "securepassword123"
      }'
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.example.com/api/v1/auth/register', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        email: 'user@example.com',
        password: 'securepassword123'
      })
    });

    const data = await response.json();
    console.log(data.token);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests

    response = requests.post(
        'https://api.example.com/api/v1/auth/register',
        json={
            'email': 'user@example.com',
            'password': 'securepassword123'
        }
    )

    data = response.json()
    print(data['token'])
    ```
  </Tab>
</Tabs>

#### Example Response

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"

```

#### Status Codes

- `201 Created` - User successfully registered
- `400 Bad Request` - Invalid email format or password length
- `409 Conflict` - Email already exists

### Login

Authenticate an existing user and receive an authentication token.

<Card title="POST /api/v1/auth/login" icon="right-to-bracket">
  Authenticate with email and password
</Card>

#### Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be a valid email format.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Must be between 6 and 72 characters.
</ParamField>

#### Response

<ResponseField name="token" type="string">
  JWT authentication token to use for subsequent API requests
</ResponseField>

#### Example Request

<Tabs>
  <Tab title="cURL">
    ```bash
    curl -X POST https://api.example.com/api/v1/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "user@example.com",
        "password": "securepassword123"
      }'
    ```
  </Tab>
  <Tab title="JavaScript">
    ```javascript
    const response = await fetch('https://api.example.com/api/v1/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        email: 'user@example.com',
        password: 'securepassword123'
      })
    });

    const data = await response.json();
    // Store the token for future requests
    localStorage.setItem('token', data.token);
    ```
  </Tab>
  <Tab title="Python">
    ```python
    import requests

    response = requests.post(
        'https://api.example.com/api/v1/auth/login',
        json={
            'email': 'user@example.com',
            'password': 'securepassword123'
        }
    )

    data = response.json()
    token = data['token']
    ```
  </Tab>
</Tabs>

#### Example Response

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c"
```

#### Status Codes

- `200 OK` - Authentication successful
- `400 Bad Request` - Invalid email format or password length
- `401 Unauthorized` - Invalid credentials

## Using the Authentication Token

After registering or logging in, include the JWT token in all subsequent API requests:

```bash
curl -X GET https://api.example.com/api/v1/templates \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

<Note>
  Tokens do not expire by default in the current implementation. Store tokens securely and never expose them in client-side code or version control.
</Note>
