## Overview

Logs allow you to track activities, changes, and observations for any row. Each log entry includes a score (0.00-10.00), a comment, and an optional event date.

**Base Path:** `/api/v1/rows/{rowId}/logs`

**Authentication:** Required - Include JWT token in `Authorization` header

## Endpoints

### Create Log

Add a new log entry to a row.

<Card title="POST /api/v1/rows/{rowId}/logs" icon="plus">
  Create a new log entry for a row
</Card>

#### Path Parameters

<ParamField path="rowId" type="UUID" required>
  The unique identifier of the row
</ParamField>

#### Request Body

<ParamField body="score" type="decimal" required>
  Numeric score for the log entry. Must be between 0.00 and 10.00 (inclusive).
</ParamField>

<ParamField body="comment" type="string" required>
  Detailed comment or description. Maximum 5000 characters. Cannot be blank.
</ParamField>

<ParamField body="eventDate" type="string">
  Optional date when the logged event occurred. Format: YYYY-MM-DD.
  
  If not provided, the current date is used.
</ParamField>

#### Response

<ResponseField name="id" type="UUID">
  Unique identifier for the log entry
</ResponseField>

<ResponseField name="score" type="decimal">
  Numeric score (0.00-10.00)
</ResponseField>

<ResponseField name="comment" type="string">
  Log entry comment
</ResponseField>

<ResponseField name="eventDate" type="ISO 8601 date">
  Date when the logged event occurred
</ResponseField>

<ResponseField name="createdAt" type="ISO 8601 timestamp">
  Timestamp when the log entry was created
</ResponseField>

#### Status Codes

- `201 Created` - Log entry successfully created
- `400 Bad Request` - Invalid request body or validation error
- `401 Unauthorized` - Missing or invalid authentication token
- `404 Not Found` - Row not found

### List Logs

Retrieve all log entries for a row.

<Card title="GET /api/v1/rows/{rowId}/logs" icon="list">
  List all log entries for a row
</Card>

#### Path Parameters

<ParamField path="rowId" type="UUID" required>
  The unique identifier of the row
</ParamField>

#### Response

Returns an array of log entry objects.

<ResponseField name="[]" type="array">
  Array of log entries
  <Expandable title="Log Object">
    <ResponseField name="id" type="UUID">
      Unique identifier for the log entry

      Numeric score (0.00-10.00)
    </ResponseField>
    <ResponseField name="comment" type="string">
      Log entry comment
    </ResponseField>
    <ResponseField name="eventDate" type="ISO 8601 date">
      Date when the logged event occurred

    <ResponseField name="createdAt" type="ISO 8601 timestamp">
      Timestamp when the log entry was created

#### Example Request

<Tabs>
  <Tab title="cURL">

  </Tab>
  <Tab title="JavaScript">

## Score Guidelines

<Note>
  The score field is flexible and can represent different metrics based on your use case:
  - **0.00-3.00**: Critical issues, poor performance, or failure
  - **3.01-5.00**: Below average, needs improvement
  - **5.01-7.00**: Acceptable, meeting minimum standards
  - **7.01-9.00**: Good performance, above average
  - **9.01-10.00**: Excellent, exceptional performance
</Note>
