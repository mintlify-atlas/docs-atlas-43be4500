## Overview

Fields are the building blocks that define the structure of your templates. Each field represents a column in your data collection, with its own data type, validation rules, and display properties. Fields give you complete control over what information you capture in each template.

<Info>
Think of fields as column definitions in a spreadsheet - they determine what kind of data each row can contain.
</Info>

## Supported Data Types

Bitácora Universal supports five flexible data types:

## Data Structure

### Database Schema

```sql
CREATE TABLE template_field (
  id            BINARY(16) NOT NULL,
  template_id   BINARY(16) NOT NULL,
  field_key     VARCHAR(60) NOT NULL,
  label         VARCHAR(120) NOT NULL,
  data_type     VARCHAR(20) NOT NULL,
  required      BOOLEAN NOT NULL DEFAULT FALSE,
  options_json  JSON NULL,
  order_index   INT NOT NULL DEFAULT 0,
  created_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
  updated_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
  PRIMARY KEY (id),
  CONSTRAINT fk_field_template
    FOREIGN KEY (template_id) REFERENCES template(id)
      ON DELETE CASCADE,
  UNIQUE KEY uk_field_template_key (template_id, field_key),
  KEY ix_field_template_order (template_id, order_index),
  CONSTRAINT ck_field_datatype
    CHECK (data_type IN ('TEXT','NUMBER','BOOLEAN','DATE','SELECT'))
);
```

<Note>
The `ON DELETE CASCADE` constraint ensures that when a template is deleted, all its fields are automatically removed.
</Note>

## Field Properties

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `id` | UUID | Yes (auto) | Unique identifier |
| `template_id` | UUID | Yes | Links field to parent template |
| `field_key` | String | Yes | Internal identifier (no spaces, unique per template) |
| `label` | String | Yes | Human-readable display name |
| `data_type` | Enum | Yes | One of: TEXT, NUMBER, BOOLEAN, DATE, SELECT |
| `required` | Boolean | Yes | Whether the field must be filled in rows |
| `options_json` | JSON | Conditional | Required for SELECT type, contains array of valid options |
| `order_index` | Integer | Yes | Display order (lower numbers appear first) |

## User Workflows

### Creating a Field

Users add fields to templates through a modal interface:

1. Navigate to a template detail page
2. Click "Añadir campo" button
3. Fill in the field configuration:
   - **fieldKey**: Internal identifier (e.g., "marca")
   - **label**: Display name (e.g., "Marca")
   - **dataType**: Choose from the five supported types
   - **required**: Toggle whether the field is mandatory
   - **options**: For SELECT type, enter comma-separated values
4. Submit to create the field

### Backend Validation

The backend enforces strict validation rules when creating fields:

```java
public FieldResponse create(UUID templateId, CreateFieldRequest req) {
    // Verify template exists and belongs to user
    templateRepository.findByIdAndOwnerId(templateId, ownerId())
            .orElseThrow(() -> new ConflictException(
                "Template no encontrado o no pertenece al usuario"
            ));

    // Prevent duplicate field keys
    String key = req.getFieldKey().trim();
    if (fieldRepository.existsByTemplateIdAndFieldKey(templateId, key)) {
        throw new ConflictException("Ya existe una columna con ese fieldKey");
    }

    // Validate data type
    String typeStr = req.getDataType().trim().toUpperCase();
    FieldType type = FieldType.valueOf(typeStr);

    // Validate SELECT options
    String optionsJson = req.getOptionsJson();
    if (type == FieldType.SELECT) {
        if (optionsJson == null || optionsJson.isBlank()) {
            throw new ConflictException(
                "SELECT requiere optionsJson (array JSON)"
            );
        }
        // Ensure it's a valid JSON array
        try {
            JsonNode node = objectMapper.readTree(optionsJson);
            if (!node.isArray()) throw new IllegalArgumentException(
                "optionsJson no es array"
            );
        } catch (Exception e) {
            throw new ConflictException(
                "optionsJson debe ser un array JSON válido, ej: [\"A\",\"B\"]"
            );
        }
    } else {
        optionsJson = null;
    }

    // Create and save field
    TemplateFieldEntity f = new TemplateFieldEntity();
    f.setTemplateId(templateId);
    f.setFieldKey(key);
    f.setLabel(req.getLabel().trim());
    f.setDataType(type.name());
    f.setRequired(req.isRequired());
    f.setOptionsJson(optionsJson);
    f.setOrderIndex(req.getOrderIndex());

    return fieldRepository.save(f);
}
```

## Data Type Validation

When users create or update rows, field values are validated against their defined types:

### TEXT Validation

```java
case TEXT -> {
    if (!(value instanceof String)) 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " debe ser TEXT"
        );
}
```

### NUMBER Validation

```java
case NUMBER -> {
    if (!(value instanceof Number)) 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " debe ser NUMBER"
        );
    // Validate it can be converted to BigDecimal
    new BigDecimal(value.toString());
}
```

### BOOLEAN Validation

```java
case BOOLEAN -> {
    if (!(value instanceof Boolean)) 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " debe ser BOOLEAN"
        );
}
```

### DATE Validation

```java
case DATE -> {
    if (!(value instanceof String)) 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " debe ser DATE (YYYY-MM-DD)"
        );
    try { 
        LocalDate.parse((String) value); 
    } catch (Exception e) { 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " DATE inválida (YYYY-MM-DD)"
        ); 
    }
}
```

### SELECT Validation

```java
case SELECT -> {
    if (!(value instanceof String)) 
        throw new ConflictException(
            "Campo " + field.getFieldKey() + " debe ser SELECT (string)"
        );
    
    String optionsJson = field.getOptionsJson();
    if (optionsJson == null || optionsJson.isBlank()) {
        throw new ConflictException(
            "Campo SELECT sin optionsJson configurado: " + field.getFieldKey()
        );
    }
    
    try {
        JsonNode arr = objectMapper.readTree(optionsJson);
        boolean found = false;
        for (JsonNode n : arr) {
            if (n.isTextual() && n.asText().equals(value)) { 
                found = true; 
                break; 
            }
        }
        if (!found) 
            throw new ConflictException(
                "Valor no permitido para " + field.getFieldKey()
            );
    } catch (ConflictException ce) {
        throw ce;
    } catch (Exception e) {
        throw new ConflictException(
            "optionsJson inválido en campo SELECT: " + field.getFieldKey()
        );
    }
}
```

## Required Field Enforcement

When `required: true`, the field must be present and non-null in all row operations:

```java
// Check during row creation
for (TemplateFieldEntity f : fields) {
    if (f.isRequired() && !values.containsKey(f.getFieldKey())) {
        throw new ConflictException(
            "Falta campo obligatorio: " + f.getFieldKey()
        );
    }
}

// Check during row updates (after merge)
for (TemplateFieldEntity f : fields) {
    if (f.isRequired()) {
        if (!current.containsKey(f.getFieldKey()) || 
            current.get(f.getFieldKey()) == null) {
            throw new ConflictException(
                "Falta campo obligatorio: " + f.getFieldKey()
            );
        }
    }
}
```
