The scoring system allows you to track quality metrics, reviews, or ratings for individual rows. Each row can have multiple log entries with scores (0-10) and detailed comments.

## Overview

Log entries provide:
- **Numeric Scoring:** Decimal scores from 0.00 to 10.00
- **Detailed Comments:** Text notes up to 5000 characters
- **Event Dating:** Optional event dates separate from creation timestamp
- **Audit Trail:** Automatic creation timestamps for tracking history

<Note>
Log entries belong to rows, not templates. Each row can have unlimited log entries to track changes over time.
</Note>

## Creating Log Entries

<Steps>

### Ensure Row Exists

Log entries are attached to rows. First, create a row in your template.

**Example Row:**

```json
{
  "id": "f1a2b3c4-d5e6-7890-abcd-ef1234567890",
  "displayName": "Toyota Corolla 2020",
  "values": {
    "marca": "Toyota",
    "kilometros": 45000
  }
}
```

### Add a Log Entry

Create a log entry with a score and comment.

**Endpoint:** `POST /api/v1/rows/{rowId}/logs`

**Basic Request:**

```json
{
  "score": 8.5,
  "comment": "Excellent condition. Engine runs smoothly, no visible damage."
}
```

**Request with Event Date:**

```json
{
  "score": 7.0,
  "comment": "Good overall, minor scratches on rear bumper. Brakes need attention.",
  "eventDate": "2026-03-05"
}
```

**Field Requirements:**
- `score`: Required, decimal 0.00-10.00, up to 2 decimal places
- `comment`: Required, non-blank, max 5000 characters
- `eventDate`: Optional, format YYYY-MM-DD

**Response (201 Created):**

```json
{
  "id": "b1c2d3e4-f5a6-7890-bcde-f12345678901",
  "score": 8.5,
  "comment": "Excellent condition. Engine runs smoothly, no visible damage.",
  "eventDate": null,
  "createdAt": "2026-03-06T14:30:00.123Z"
}
```

**With Event Date:**

```json
{
  "id": "c2d3e4f5-a6b7-8901-cdef-123456789012",
  "score": 7.0,
  "comment": "Good overall, minor scratches on rear bumper. Brakes need attention.",
  "eventDate": "2026-03-05",
```

**Implementation Reference:** `LogService.java:29`

```java
public LogResponse create(UUID rowId, CreateLogRequest req) {
    // Verify row exists and belongs to user
    boolean exists = rowRepository.existsById(rowId);
    if (!exists) throw new ConflictException("Row no encontrada");
    
    // Parse optional event date
    LocalDate eventDate = null;
    if (req.getEventDate() != null && !req.getEventDate().isBlank()) {
        try { 
            eventDate = LocalDate.parse(req.getEventDate().trim()); 
        } catch (Exception e) { 
            throw new ConflictException("eventDate inválida (YYYY-MM-DD)"); 
        }
    }
    
    // Create log entry
    LogEntryEntity e = new LogEntryEntity();
    e.setOwnerId(ownerId());
    e.setRowId(rowId);
    e.setScore(req.getScore());
    e.setComment(req.getComment().trim());
    e.setEventDate(eventDate);
    
    LogEntryEntity saved = logRepository.save(e);
    return new LogResponse(...);
}
```

## Score Validation

### Valid Scores

Scores must be between 0.00 and 10.00 with up to 2 decimal places.

**Valid Examples:**

```json
{"score": 0.00}    // Minimum
{"score": 5}       // Integer
{"score": 7.5}     // One decimal
{"score": 8.75}    // Two decimals
{"score": 10.00}   // Maximum
```

### Invalid Scores

**Below Minimum:**

```json
{"score": -1.0}
```

**Error:** `400 Bad Request` - Validation error: must be >= 0.00

**Above Maximum:**

```json
{"score": 10.5}
```

**Error:** `400 Bad Request` - Validation error: must be less than or equal to 10.00

**Database Constraint:** `V1__init.sql:61`

```sql
CONSTRAINT ck_log_score_range
    CHECK (score >= 0.00 AND score <= 10.00)
```

### Request DTO Validation

**Reference:** `CreateLogRequest.java:8`

```java
public class CreateLogRequest {
    @NotNull
    @DecimalMin(value = "0.00")
    @DecimalMax(value = "10.00")
    private BigDecimal score;
    
    @NotBlank
    @Size(max = 5000)
    private String comment;
    
    // Optional, format "YYYY-MM-DD"
    private String eventDate;
}
```

## Event Date vs Created At

Understand the difference between `eventDate` and `createdAt`:

<CardGroup cols={2}>

<Card title="eventDate" icon="calendar-day">
  **Optional** - When the rated event occurred  
  **Format:** YYYY-MM-DD  
  **Use:** Backdating reviews or tracking historical events  
  **Example:** "2026-03-05"
</Card>

<Card title="createdAt" icon="clock">
  **Automatic** - When the log entry was created  
  **Format:** ISO 8601 timestamp  
  **Use:** Audit trail of when reviews were submitted  
  **Example:** "2026-03-06T14:30:00.123Z"
</Card>

</CardGroup>

## Database Schema

### Log Entry Table

**Reference:** `V1__init.sql:48`

```sql
CREATE TABLE log_entry (
    id            BINARY(16) NOT NULL,
    owner_id      VARCHAR(64) NOT NULL,
    row_id        BINARY(16) NOT NULL,
    score         DECIMAL(4,2) NOT NULL,
    comment       TEXT NOT NULL,
    event_date    DATE NULL,
    created_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    PRIMARY KEY (id),
    CONSTRAINT fk_log_row
        FOREIGN KEY (row_id) REFERENCES object_row(id)
            ON DELETE CASCADE,
    KEY ix_log_owner_row_created (owner_id, row_id, created_at),
    CONSTRAINT ck_log_score_range
        CHECK (score >= 0.00 AND score <= 10.00)
);
```

### Key Features

**Cascade Delete:**
When a row is deleted, all its log entries are automatically deleted.

```sql
CONSTRAINT fk_log_row
    FOREIGN KEY (row_id) REFERENCES object_row(id)
        ON DELETE CASCADE
```

**Indexed Queries:**
Fast retrieval of logs by owner and row:

```sql
KEY ix_log_owner_row_created (owner_id, row_id, created_at)
```

**Score Constraint:**
Database-level validation:

```sql
CONSTRAINT ck_log_score_range
    CHECK (score >= 0.00 AND score <= 10.00)
```

## Best Practices

<Tip>
**Use Decimal Precision:** Scores support 2 decimal places. Use `8.75` instead of `9` for more granular ratings.
</Tip>

<Tip>
**Meaningful Comments:** Provide detailed, actionable feedback in comments rather than generic statements.
</Tip>

<Tip>
**Consistent Event Dates:** When backdating entries, always specify `eventDate` to maintain accurate historical records.
</Tip>

<Warning>
Log entries cannot be edited or deleted. Create a new entry if you need to update information.
</Warning>
