Templates are the foundation of Bitácora Universal. They define the structure of your data, similar to creating tables in a database or sheets in Excel.

## Overview

A template consists of:
- **Template Metadata:** Name and description
- **Fields (Columns):** Typed data fields with validation rules
- **Rows (Data):** Individual records conforming to the template structure

<Note>
Each user's templates are completely isolated. Templates are identified by `owner_id` and `name` uniquely.
</Note>

## Creating Your First Template

<Steps>

### Create a Template

Start by creating a template to define your data category.

**Endpoint:** `POST /api/v1/templates`

**Request:**

```json
{
  "name": "Coches Usados",
  "description": "Mi excel personal para registrar coches"
}
```

**Validation Rules:**
- `name`: Required, max 120 characters, must be unique per user
- `description`: Optional, max 500 characters

**Response (201 Created):**

```json
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Coches Usados",
  "description": "Mi excel personal para registrar coches",
  "createdAt": "2026-03-06T10:30:00.123Z",
  "updatedAt": "2026-03-06T10:30:00.123Z"
}
```

<Warning>
Attempting to create a template with a duplicate name returns `409 Conflict`: "Ya existe una plantilla con ese nombre"
</Warning>

**Implementation Reference:** `TemplateService.java:32`

```java
public TemplateResponse create(CreateTemplateRequest req) {
    String owner = ownerId();
    String name = req.getName().trim();
    
    if (repository.existsByOwnerIdAndName(owner, name)) {
        throw new ConflictException("Ya existe una plantilla con ese nombre");
    }
    
    TemplateEntity e = new TemplateEntity();
    e.setOwnerId(owner);
    e.setName(name);
    e.setDescription(req.getDescription() == null ? null : req.getDescription().trim());
    
    TemplateEntity saved = repository.save(e);
    return new TemplateResponse(...);
}
```

### Add Fields to Your Template

Define the columns/fields for your template. Each field has a specific data type and validation rules.

**Endpoint:** `POST /api/v1/templates/{templateId}/fields`

**Supported Data Types:**
- `TEXT`: String values
- `NUMBER`: Numeric values (integers or decimals)
- `BOOLEAN`: true/false values
- `DATE`: Date in YYYY-MM-DD format
- `SELECT`: Predefined options (dropdown)

**Example: Add a TEXT field**

```json
{
  "fieldKey": "marca",
  "label": "Marca del coche",
  "dataType": "TEXT",
  "required": true,
  "orderIndex": 0
}
```

**Example: Add a SELECT field**

```json
{
  "fieldKey": "combustible",
  "label": "Tipo de combustible",
  "dataType": "SELECT",
  "required": true,
  "optionsJson": "[\"Gasolina\",\"Diesel\",\"Eléctrico\",\"Híbrido\"]",
  "orderIndex": 1
}
```

**Example: Add a NUMBER field**

```json
{
  "fieldKey": "kilometros",
  "label": "Kilómetros",
  "dataType": "NUMBER",
  "required": false,
  "orderIndex": 2
}
```

**Example: Add a DATE field**

```json
{
  "fieldKey": "fecha_compra",
  "label": "Fecha de compra",
  "dataType": "DATE",
  "required": false,
  "orderIndex": 3
}
```

**Response (201 Created):**

```json
{
  "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "fieldKey": "marca",
  "label": "Marca del coche",
  "dataType": "TEXT",
  "required": true,
  "optionsJson": null,
  "orderIndex": 0
}
```

<Tip>
Use `orderIndex` to control the display order of fields. Fields are returned sorted by this value.
</Tip>

## Field Types in Detail

### TEXT Fields

Store string values of any length.

```json
{
  "fieldKey": "notes",
  "label": "Notas adicionales",
  "dataType": "TEXT",
  "required": false,
  "orderIndex": 10
}
```

**Example Values:**
```json
{
  "notes": "Coche en excelente estado"
}
```

### NUMBER Fields

Store integers or decimal values.

```json
{
  "fieldKey": "precio",
  "label": "Precio en euros",
  "dataType": "NUMBER",
  "required": true,
  "orderIndex": 5
}
```

**Example Values:**
```json
{
  "precio": 25000.50
}
```

**Validation:** `RowService.java:222`

```java
case NUMBER -> {
    if (!(value instanceof Number)) 
        throw new ConflictException("Campo " + field.getFieldKey() + " debe ser NUMBER");
    new BigDecimal(value.toString());
}
```

### BOOLEAN Fields

Store true/false values.

```json
{
  "fieldKey": "negociable",
  "label": "Precio negociable",
  "dataType": "BOOLEAN",
  "required": false,
  "orderIndex": 6
}
```

**Example Values:**
```json
{
  "negociable": true
}
```

### DATE Fields

Store dates in `YYYY-MM-DD` format.

```json
{
  "fieldKey": "ultima_revision",
  "label": "Última revisión",
  "dataType": "DATE",
  "required": false,
  "orderIndex": 7
}
```

**Example Values:**
```json
{
  "ultima_revision": "2026-01-15"
}
```

**Validation:** `RowService.java:230`

```java
case DATE -> {
    if (!(value instanceof String)) 
        throw new ConflictException("Campo " + field.getFieldKey() + " debe ser DATE (YYYY-MM-DD)");
    try { 
        LocalDate.parse((String) value); 
    } catch (Exception e) { 
        throw new ConflictException("Campo " + field.getFieldKey() + " DATE inválida (YYYY-MM-DD)"); 
    }
}
```

### SELECT Fields

Store predefined dropdown values.

```json
{
  "fieldKey": "estado",
  "label": "Estado del vehículo",
  "dataType": "SELECT",
  "required": true,
  "optionsJson": "[\"Nuevo\",\"Seminuevo\",\"Usado\"]",
  "orderIndex": 4
}
```

**Example Values:**
```json
{
  "estado": "Usado"
}
```

<Warning>
For SELECT fields, the value must exactly match one of the options in `optionsJson`. Case-sensitive.
</Warning>

## Database Schema

### Template Table

**Reference:** `V1__init.sql:1`

```sql
CREATE TABLE template (
    id            BINARY(16) NOT NULL,
    owner_id      VARCHAR(64) NOT NULL,
    name          VARCHAR(120) NOT NULL,
    description   VARCHAR(500) NULL,
    created_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    PRIMARY KEY (id),
    UNIQUE KEY uk_template_owner_name (owner_id, name)
);
```

### Template Field Table

**Reference:** `V1__init.sql:12`

```sql
CREATE TABLE template_field (
    id            BINARY(16) NOT NULL,
    template_id   BINARY(16) NOT NULL,
    field_key     VARCHAR(60) NOT NULL,
    label         VARCHAR(120) NOT NULL,
    data_type     VARCHAR(20) NOT NULL,
    required      BOOLEAN NOT NULL DEFAULT FALSE,
    options_json  JSON NULL,
    order_index   INT NOT NULL DEFAULT 0,
    created_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    updated_at    TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    PRIMARY KEY (id),
    CONSTRAINT fk_field_template
        FOREIGN KEY (template_id) REFERENCES template(id)
            ON DELETE CASCADE,
    UNIQUE KEY uk_field_template_key (template_id, field_key),
    CONSTRAINT ck_field_datatype
        CHECK (data_type IN ('TEXT','NUMBER','BOOLEAN','DATE','SELECT'))
);
```

## Complete Example: Car Inventory Template

```bash
# 1. Create template
curl -X POST 'http://localhost:8080/api/v1/templates' \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "Inventario de Coches",
    "description": "Seguimiento de vehículos en stock"
  }'

# Save the template ID from response
TEMPLATE_ID="a1b2c3d4-e5f6-7890-abcd-ef1234567890"

# 2. Add brand field
curl -X POST "http://localhost:8080/api/v1/templates/$TEMPLATE_ID/fields" \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "fieldKey": "marca",
    "label": "Marca",
    "dataType": "TEXT",
    "required": true,
    "orderIndex": 0
  }'

# 3. Add fuel type field
curl -X POST "http://localhost:8080/api/v1/templates/$TEMPLATE_ID/fields" \
  -H 'Authorization: Bearer YOUR_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "fieldKey": "combustible",
    "label": "Combustible",
    "dataType": "SELECT",
    "required": true,
    "optionsJson": "[\"Gasolina\",\"Diesel\",\"Eléctrico\"]",
    "orderIndex": 1

# 4. Add price field
curl -X POST "http://localhost:8080/api/v1/templates/$TEMPLATE_ID/fields" \

curl -X GET "http://localhost:8080/api/v1/templates/$TEMPLATE_ID" \
  -H 'Authorization: Bearer YOUR_TOKEN'
```

## Frontend Integration Example

**Reference:** `NewTemplateModal.tsx:17`

```typescript
async function submit() {
  setError(null);
  
  const n = name.trim();
  const d = description.trim();
  
  if (!n) {
    setError("El nombre es obligatorio.");
    return;
  }
  
  setSaving(true);
  try {
    await api.post("/api/v1/templates", {
      name: n,
      description: d ? d : null,
    });
    
    setName("");
    setDescription("");
    props.onClose();
    props.onCreated();
  } catch (e: any) {
    setError(e?.message ?? "No se pudo crear la plantilla");
  } finally {
    setSaving(false);
  }
}
```
